# Research Contract Compiler

The contract compiler turns a governed research route into an executable contract.

Reviewable responsibilities:

- resolve WorkOrder, route authority, application owner, and allowed stage into an executable contract
- bind required inputs, source manifests, feature/label scope, and eligible Layer 2 engines
- compile invariants and failure modes for leakage, fold policy, cache identity, replay completeness, and runtime posture
- define allowed claims, blocked claims, and diagnostic-only outputs before computation
- declare artifact outputs, evidence envelopes, and gate inputs expected at completion
- fail closed when a route, owner, source, engine, or validation requirement is missing
