# Work Orders

Work orders define the authorized research request before any application or engine runs.

Reviewable responsibilities:

- objective, owner, requested stage, route candidate, and review authority
- allowed next action derived from stage, evidence state, and governance policy
- explicit non-goals, blocked claims, runtime posture, and required evidence path
- handoff into Layer 4 research applications through a compilable route contract
- rejection when the request lacks owner, scope, evidence path, or permitted stage
